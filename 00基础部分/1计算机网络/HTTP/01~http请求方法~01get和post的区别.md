## get 与 post的区别
1. 作用
  * get一般用于资源获取，不会影响资源的状态
  * post一般用于修改服务器上的资源
> 所以GET在浏览器回退时是无害的，而POST会再次提交请求。

2. 传递数据的方式
  * get使用URL传递数据
  * post一般使用request body传递数据

3. 缓存：get能缓存，post不能
4. 历史记录：get请求参数会被完整保留在浏览器历史记录里，而POST中的参数不会被保留
5. 安全： get将参数显示在浏览器地址栏，不能用来传递敏感数据

> HTTP 协议没有 Body 和 URL 的长度限制，对 URL 限制的大多是浏览器和服务器的原因
